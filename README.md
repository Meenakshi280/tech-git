# ✅ My Tasks — my first real app

A tiny to-do app I built to understand **how software actually works**, end to end.

## What it does
Type a task → click **Add**. Click a task to mark it **done**. Click **✕** to delete.
It even **remembers** your tasks after you close the tab.

## The 3 layers every app is made of
Software feels like magic until you see it's just three cooperating parts:

| File         | Layer        | Plain-English job                          |
|--------------|--------------|--------------------------------------------|
| `index.html` | **Structure**| WHAT is on screen (title, box, button, list) |
| `style.css`  | **Looks**    | HOW it looks (colours, spacing, fonts)     |
| `app.js`     | **Behavior** | WHAT happens when you click (+ remembers data) |

> A useful way to think about it: **HTML is the skeleton, CSS is the clothing, JavaScript is the brain.**

## How to run it
Just double-click `index.html` — it opens in your browser. That's the whole app running on your computer.

## The complete software flow (this is the PM part 👇)
1. **Idea** → "people need a simple to-do list"
2. **Build** → write the 3 files above
3. **Run locally** → test it on your own machine
4. **Version control** → save snapshots with Git (`add` → `commit` → `push`)
5. **Ship / Deploy** → put it live on the internet so real users can use it
6. **Improve** → get feedback, add features, repeat

Every product you'll ever manage follows this same loop.
